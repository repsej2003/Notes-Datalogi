
# analyse af algortimer
## Korrekthed
- stopper for alle input
- Giver det rigtigt svar når den stopper
## Kvaliteter
- Hatsighed
- Pladsforbrug
- Kompleksitet af implementation
- ekstra egenskaber (problemspeciffker)

Først korrekthed så kvaliteter, ofte hastighed


### ting nødvendigt for analyse
1. Model af problemer
2. Model af maskine (Ram model, simpel forståelse af computer)
3. Mål for kvalitet - tidsforbrug
4. mateatiske værktøjer

## Måle tidsforbrug
For en givent størrelse af n af input

hastigheden kan være forkelige til forskellige input

Så vi kan snakke om
- Worstcase (Er ofte repastiv for average case, kan være pessimist)
- Average case (Svær matematisk, svært at vide hvad sandsynligheden er)
- Best case (Ikke så relvant)

Tides forbrug stiger når størrelse af input stiger. 

Så analyse resluter en en funktion, der siger worst case for et givet n input
To kan samlignes, den der bliver mindst når n bliver stor nok er bedst

asymptotisk voksehatighed at sammenligne disse funktioner
